> **לקלוד:** קרא קובץ זה במלואו בתחילת כל שיחה חדשה כדי להבין את ההקשר של הפרויקט.

---

## 💡 מהות הפרויקט (TL;DR)

**The Crawler** = פלטפורמת **SaaS** שאוספת ידיעות ומידע ממגוון **עצום** של מקורות, **מסננת לפי איכות**, ומציגה **תובנות ממוקדות שוק הון** שעוזרות למשתמש **לקבל החלטות השקעה**.

- האיכות נמדדת ב**סינון** ובטיב החומרים.
- התוצאה נמדדת ב**הצגת המידע ללקוח** בצורה שמסייעת להחלטות.
- היתרון: **ריבוי מקורות** + **תיעדוף חכם** (כי יש מגוון עצום).
- שם עבודה: **NewFin** (זמני). דומיין מתוכנן: **newfin.il**. אירוח: **נטליפיי** (כמו danino-app, יש מנוי), עם אפשרות לחבר דומיין ‎.il בהמשך.

## ⚖️ המסגרת המשפטית (קריטי — לב המוצר)

> זה מה שמבדיל בין עסק שמוכרים במיליונים לעסק שנסגר במכתב אחד מעו"ד.

- **עובדות** = חופשיות לכולם (מי אמר מה, מספרים, אירועים). **מותר לדווח** + לאזכר מקור + קישור.
- **ניסוח / ניתוח / דעה / גרף** של מישהו אחר = **מוגן בזכויות יוצרים**. אסור להעתיק, גם עם אזכור.
- **לעולם לא** להפיץ דוח מורשה/בתשלום (FactSet, Bloomberg, מחקר בנקים). רישיון אישי ≠ זכות הפצה. זה רעל לסקייל ולגיוס.
- **הדרך הנכונה:** קוראים את הדוח → כותבים **בניסוח שלנו** את המסקנה + קישור למקור פומבי. מודל עיתונאי.
- **גרפים:** לא להעתיק — לבנות **גרף חדש** מנתונים ציבוריים (מחירים/מדדים פתוחים), אוטומטית מ-API.
- **בעתיד:** רישיון הפצה רשמי (Benzinga, Morningstar, Zacks) כשיש הכנסה.
- צריך עמוד **מדיניות עריכה** (Editorial Policy) — אמון + הגנה משפטית + SEO.

ראה `docs/04-tech-stack-sources.md` ו-`docs/01-vision-strategy-legal.md` לפרטים.

## 🧭 בידול מ-FocusFin (המתחרה הישיר)

FocusFin (dash.focusfin.co.il) כבר עושה קרולר פיננסי ממוקד שוק הון ישראלי. עשינו רֶקוֹן מלא (ראה `docs/03-competitive-teardown-focusfin.md`). היתרונות שלנו:

1. **ציון מהותיות** — להם אין! הפיד שלהם רועש ("שינוי כתובת" באותו גודל כמו "דירקטור מכר מניות"). אצלנו הרעש שוקע.
2. **טון עיתונאי** עקבי: עובדה → הקשר → שורה תחתונה (בלי גוף ראשון).
3. **Deep-links** ישירים להודעה (הם מקשרים לדף החברה).
4. **חומה היברידית** — שכבה ציבורית מאונדקסת ל-SEO + הרשמה לפיד מלא (הם חוסמים הכל → מאבדים טראפיק אורגני).
5. **דפי חברה/תגית כעמודי נחיתה** — מאות עמודים מאונדקסים (זהב SEO שהם משאירים על השולחן).
6. **זיקוק** של 100 מקורות לתובנה אחת — לא "עוד ערוץ שמשכפל".

## ✍️ ה-DNA של הכתיבה

> **כותרת עובדתית → פסקת דיווח (עובדות בלבד) → שורה תחתונה (הקשר/משמעות, בלי גוף ראשון)**

- טון: יבש, ענייני, סמכותי — כמו כלכליסט / Bloomberg.
- ה"שורה התחתונה" היא **החתימה של המוצר** — מה שהופך דיווח יבש לתובנה.
- ה-AI מפיק את זה אוטומטית מ-**prompt סגנון קבוע**.

## 🧱 Stack טכני (מתוכנן)

- **Frontend:** React 18 + TypeScript + Vite + Tailwind RTL (כמו danino-app)
- **Backend:** Supabase (Postgres + Auth + Edge Functions + pgvector)
- **Auth:** Supabase Auth — Google/Microsoft/Apple/Email (חינם)
- **Deploy:** Netlify (חינם, אוטומטי מ-GitHub)
- **Cron:** Supabase cron / GitHub Actions (חינם)
- **AI:** Groq (חינם, לתיוג/סיווג זול) + Claude (לסיכומים עמוקים)
- **מקורות נתונים:** Maya API (`mayaapi.tase.co.il`), RSS, Finnhub, FRED, Yahoo, EDGAR, Alpha Vantage

עיקרון עלות: **סינון זול קודם** (כללים/embeddings/מודל חינמי) → רק מה שעבר סף מגיע ל-LLM יקר. **אגרגציה לפי משתמש, לא לפי ידיעה.**
